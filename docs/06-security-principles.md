# 06 - 安全原则

> 本文明确平台的安全默认原则，覆盖权限、隔离、密钥、审计与故障隔离。
> 安全要求优先于审美与效率，是接入协议与 SDK 设计的硬约束。

## 1. 最小权限

- 用户与服务身份只授予完成任务所需的最小权限；
- 项目级 RBAC 先行（owner/maintainer/developer/viewer），按需才扩展 ABAC；
- 平台管理员权限与项目权限分离，管理员不属于任何项目但受平台级审计。

## 2. 项目隔离与环境隔离

- 数据访问默认按 `project_id` 隔离，查询必须带项目上下文并校验权限；
- 环境隔离：`dev` / `staging` / `prod` 配置与密钥独立，禁止跨环境默认共享；
- 可观测性数据查询通过带权限上下文的跳转或代理，确保只能看到有权限项目的数据。

## 3. 用户身份与服务身份分离

- 用户身份用于人在管理后台操作；
- 服务身份（ServiceCredential）用于接入项目调用平台 API；
- 两者凭证独立，服务凭证可轮换、可吊销；
- **禁止不同项目共享管理员凭证或服务凭证**。

## 4. 平台故障隔离

SDK、CLI 与接入项目必须保证平台故障不拖垮业务系统。当前 TS/Python SDK 已对平台 API 调用设置超时；运行时级熔断、异步缓冲和丢弃策略属于后续 SDK 增强，不应由业务主流程同步等待平台：

- **超时**：所有平台调用设置合理超时，不无限等待；
- **熔断**：连续失败触发熔断，快速失败；
- **降级**：平台不可用时，业务可继续运行（如日志降级为本地输出）；
- **异步上报**：可观测性数据异步上报，不阻塞主流程；
- **本地缓冲**：上报失败时本地缓冲，恢复后补发；
- **丢弃策略**：缓冲满后按策略丢弃旧数据，保护业务内存；
- **重试上限**：限制重试次数与退避，避免雪崩；
- **敏感信息过滤**：上报前过滤密钥、Token、密码等敏感字段。

## 5. 密钥管理

- 密钥真实值**不进入代码仓库、不进入数据库、不进入日志**；
- 平台只存 `SecretMetadata`（引用、版本、轮换策略），真实值在外部 Secret Store / KMS；
- `project.yaml` 与所有仓库文件禁止写入密钥、Token、密码、含密码连接字符串；
- 凭证支持轮换与吊销，轮换事件入审计；
- 检测到疑似密钥模式时拒绝提交并提示。

## 6. 敏感字段脱敏

- 审计事件 payload 必须脱敏，移除密钥/Token/密码字段；
- 日志记录不包含 Authorization 头、密码、Token、会话标识；
- 调试输出前检查并脱敏敏感字段。

## 7. 审计独立

- AuditEvent 独立存储，不与业务表强外键耦合；
- 只增不改，保留周期按策略；
- 高风险操作（密钥轮换、权限变更、项目删除/归档、凭证吊销）必须记录审计；
- 审计可追溯人员变更（项目负责人变更留痕）。

## 8. Token 轮换与吊销

- 服务凭证与用户 Token 支持轮换与吊销；
- 吊销即时生效，轮换不中断服务（新旧凭证过渡期）；
- 过期凭证自动失效。

## 9. 外部输入校验

- 所有外部输入（用户输入、API 响应、URL 参数、Webhook）使用前必须校验；
- 不将未校验输入直接拼接进 SQL、命令行或 HTML；
- 文件路径输入检查路径遍历（`../` 等）。

## 10. 数据保留与删除

- 涉及删除数据或数据库迁移时，先说明影响范围与风险；
- 数据库 schema 变更必须可回滚；
- 日志/指标/trace 保留周期按策略配置，避免成本失控；
- 项目归档不立即物理删除，保留可追溯期。

## 11. 禁止共享管理员凭证

- 不同项目不共享管理员凭证；
- 平台级管理员操作与项目级操作权限分离；
- 任何凭证不硬编码到代码或配置。
