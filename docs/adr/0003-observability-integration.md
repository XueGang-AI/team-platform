# ADR-0003 可观测性集成而非重造

- **状态**：accepted
- **日期**：2026-06-17

## 背景

平台需要为接入项目提供统一的日志、指标、链路追踪能力。可选路径：自研采集与存储，或集成成熟开源组件。

自研意味着重写日志存储、时序数据库、链路存储与可视化，投入巨大且无差异化价值，安全与运维成本远超收益。

## 决策

**集成成熟开源组件，不自研存储与可视化**：

- **采集**：接入方用 OpenTelemetry SDK 采集，注入统一维度标签；
- **日志存储**：Loki；
- **指标存储**：Prometheus；
- **链路存储**：Tempo；
- **可视化**：Grafana；
- **平台职责**：接收采集数据并关联项目维度，存索引与跳转链接到 PostgreSQL，提供带项目上下文的 Grafana 跳转/嵌入。

## 理由

- **避免重造基础设施**：这些组件是业界标准，自研无竞争优势；
- **生态成熟**：OTel + Prometheus + Loki + Tempo + Grafana 是事实标准，文档与社区支持充分；
- **平台聚焦差异化**：平台价值在统一关联与治理，不在存储引擎；
- **可替换**：通过适配层抽象，未来可替换具体组件。

替代方案「自研可观测性存储」不选：投入产出比极低，且难以达到成熟组件的可靠性。

## 影响

- 平台依赖这些外部组件的可用性与运维；
- 必须规范维度标签，保证跨组件关联一致；
- Dashboard 体验部分依赖 Grafana，平台提供跳转与嵌入而非完全替代；
- 需在 Phase 1 通过 Docker Compose 标准化编排这些组件。
